## 💼 管理员功能

### 1. 申请列表查看
- 📋 查看所有用户提交的认证申请
- 🔍 显示申请人姓名、业务模式、身份类型
- ⏰ 显示提交时间
- 🎨 不同状态用不同颜色标识：
  - 🟡 **待审核** - 黄色
  - 🟢 **已通过** - 绿色
  - 🔴 **已驳回** - 红色

### 2. 申请详情查看
点击任意申请可查看：
- 📝 申请人完整信息
- 🏢 业务模式和身份类型
- 📄 所有表单字段内容
- 📅 提交时间
- 📷 上传的图片（如有）

### 3. 审批操作

#### ✅ 通过申请
1. 在申请详情页面
2. 点击绿色 **"✅ 通过"** 按钮
3. 确认操作
4. ✅ 申请状态变更为"已通过"
5. 用户登录后可直接进入对应产品后台

#### ❌ 驳回申请
1. 在申请详情页面
2. 点击红色 **"❌ 驳回"** 按钮
3. 在弹窗中填写驳回原因（必填）
4. 确认提交
5. ✅ 申请状态变更为"已驳回"
6. 用户登录后可查看驳回原因并重新提交

## 🔄 工作流程

### 完整审核流程

```
用户提交申请
    ↓
管理员收到待审核申请（黄色标识）
    ↓
点击查看详情 → 审核资料
    ↓
判断：资料是否符合要求？
    ↓
    ├─ 是 → 点击"通过" → 用户获得权限 ✅
    └─ 否 → 点击"驳回" → 填写原因 → 用户收到通知 ❌
         ↓
    用户修改后重新提交
```

### 申请状态说明

| 状态 | 标识 | 说明 | 后续操作 |
|------|------|------|---------|
| **待审核** | 🟡 | 用户已提交，等待管理员审核 | 管理员可通过/驳回 |

| **已驳回** | 🔴 | 管理员拒绝，需修改 | 用户可重新提交 |

## 🔒 权限说明

### 管理员与普通用户的区别

| 功能 | 管理员 | 普通用户 |
|------|--------|---------|
| **登录后界面** | 管理员审核后台 | 业务模式选择/审核状态页 |
| **查看所有申请** | ✅ 可以 | ❌ 只能看自己的 |
| **审批申请** | ✅ 可以 | ❌ 不可以 |
| **提交认证表单** | ❌ 不需要 | ✅ 需要 |
| **进入产品后台** | ❌ 仅审核 | ✅ 审核通过后可以 |

### 角色识别机制

用户数据结构中的 `role` 字段决定权限：
```javascript
{
  "email": "admin@test.com",
  "password": "admin1234",
  "role": "admin"  // "admin" 或 "user"
}
```

- **role = "admin"**：登录后显示管理员后台
- **role = "user"**：登录后显示普通用户流程

## ⚠️ 注意事项

1. **管理员账号不能提交认证申请**
   - 管理员账号专用于审核
   - 如需测试用户流程，请使用普通用户账号

2. **审批操作不可撤销**
   - 通过/驳回后无法直接撤回
   - 如需修改，需手动在localStorage中调整

3. **驳回原因必须填写**
   - 帮助用户了解问题所在
   - 提高重新提交的成功率

4. **退出登录**
   - 点击右上角"退出登录"按钮
   - 返回登录页面
